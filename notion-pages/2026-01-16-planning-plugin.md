# 2026.01.16 Planning Plugin - Ralph-loop 기반 아이디어 생성 플러그인

## 개요

Planning Plugin은 Ralph-loop 패턴으로 앱 개발 아이디어를 반복 생성하고 시장성 평가를 통해 필터링하는 기획 플러그인입니다.

## 1. 플러그인 소개

### 주요 특징

- Ralph-loop 아키텍처: Stop Hook이 루프의 중앙 컨트롤러
- 세션 기반 관리: 매 실행마다 새 폴더 생성, 결과 분리 보관
- 병렬 파이프라인: 3개의 에이전트가 동시에 다른 분야 탐색
- 웹 검색 기반 니즈 파악: 실제 사용자 불만과 니즈 조사
- 시장성 기반 평가: 100점 만점 기준으로 각 아이디어 평가
- 파일 분리 관리: 통과된 아이디어는 개별 파일로 저장

## 2. Ralph-loop 아키텍처

### 핵심 원리

```
┌─────────────────────────────────────────────────────────┐
│                    Ralph-loop 패턴                       │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Command (초기화만)     Stop Hook (루프 컨트롤러)         │
│        │                      │                          │
│        ▼                      │                          │
│  ┌──────────┐                 │                          │
│  │state.md  │                 │                          │
│  │생성      │                 │                          │
│  └────┬─────┘                 │                          │
│       │                       │                          │
│       ▼ (종료 시도)           │                          │
│       ├───────────────────────▶                          │
│       │                  ┌────┴─────┐                    │
│       │                  │진행 확인  │                    │
│       │                  │목표 미달? │                    │
│       │                  └────┬─────┘                    │
│       │                       │ block + Task 지시        │
│       ◀───────────────────────┤                          │
│       │                       │                          │
│       ▼                       │                          │
│  ┌──────────┐                 │                          │
│  │Task 실행 │                 │                          │
│  │(병렬)    │                 │                          │
│  └────┬─────┘                 │                          │
│       │                       │                          │
│       ▼ (종료 시도)           │                          │
│       ├───────────────────────▶                          │
│       │                  ┌────┴─────┐                    │
│       │                  │목표 달성? │──▶ allow (종료)    │
│       │                  └────┬─────┘                    │
│       │                       │ block (반복)             │
│       ◀───────────────────────┘                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 장점

| 기존 방식 | Ralph-loop 방식 |
|----------|----------------|
| 커맨드에서 반복 제어 | Stop Hook이 반복 제어 |
| 지시가 여러 곳에 분산 | 지시가 Hook에 집중 |
| 복잡한 상태 관리 | 단순한 상태 머신 |

## 3. 설치 방법

```bash
claude /plugin add [plugin-path]
```

또는 `--plugin-dir` 옵션으로 직접 사용:

```bash
claude --plugin-dir /path/to/planning-plugin
```

---

## 4. 사용 가능한 명령어

### 아이디어 생성

```bash
# 기본 실행 (100개 목표, 80점 이상 합격)
/idea-generate

# 옵션 지정
/idea-generate --target 50 --threshold 75
/idea-generate --domain 헬스케어 --target 30
```

### 옵션

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--target <n>` | 목표 합격 아이디어 개수 | 100 |
| `--threshold <score>` | 합격 점수 기준 (0-100) | 80 |
| `--max-iterations <n>` | 최대 반복 횟수 | 무제한 |
| `--domain <domain>` | 특정 분야 지정 | 웹 검색 발굴 |

### 아이디어 고도화

```bash
/idea-refine --top 10
/idea-refine --top 5 --threshold 85
```

### 루프 취소

```bash
/cancel-idea-loop
```

## 5. 병렬 실행 구조

### 아이디어 생성: 3개 파이프라인 동시

```
                    Stop Hook 지시
                         │
       ┌─────────────────┼─────────────────┐
       │                 │                 │
       ▼                 ▼                 ▼
┌───────────────┐ ┌───────────────┐ ┌───────────────┐
│  Pipeline P1  │ │  Pipeline P2  │ │  Pipeline P3  │
│   (분야 A)    │ │   (분야 B)    │ │   (분야 C)    │
├───────────────┤ ├───────────────┤ ├───────────────┤
│ 1.분야 발굴   │ │ 1.분야 발굴   │ │ 1.분야 발굴   │
│ 2.시장 조사   │ │ 2.시장 조사   │ │ 2.시장 조사   │
│ 3.아이디어생성│ │ 3.아이디어생성│ │ 3.아이디어생성│
│ 4.평가 & 저장 │ │ 4.평가 & 저장 │ │ 4.평가 & 저장 │
└───────────────┘ └───────────────┘ └───────────────┘
```

### 병렬 처리 이점

| 항목 | 순차 처리 | 병렬 처리 (3x) |
|------|----------|---------------|
| 1회 반복당 분야 | 1개 | 3개 |
| 1회 반복당 아이디어 | 5-10개 | 15-30개 |
| 100개 목표 달성 | ~15회 | ~5회 |

## 6. 에이전트 구성

### 메인 에이전트

| 에이전트 | 역할 | 병렬 실행 |
|----------|------|----------|
| `idea-pipeline` | 분야 발굴→조사→생성→평가 전체 파이프라인 | 3개 동시 |
| `idea-refiner` | 아이디어 상세 고도화 | 5개씩 배치 |

### 보조 에이전트 (참조용)

| 에이전트 | 역할 |
|----------|------|
| `domain-explorer` | 분야 발굴 |
| `market-researcher` | 시장 조사 |
| `idea-generator` | 아이디어 생성 |
| `idea-evaluator` | 시장성 평가 |

## 7. 평가 기준

| 기준 | 점수 | 평가 요소 |
|------|------|----------|
| 시장 규모 | 0-25 | 잠재 사용자 수, TAM/SAM/SOM, 성장률 |
| 경쟁 환경 | 0-25 | 기존 앱 대비 차별화, 진입 장벽, 경쟁 강도 |
| 실현 가능성 | 0-25 | 기술 난이도, 개발 비용, MVP 구현 용이성 |
| 타이밍 | 0-25 | 시장 성숙도, 트렌드 부합, 사용자 준비도 |

## 8. 세션 관리

매번 `/idea-generate` 실행 시 새로운 세션 폴더가 생성됩니다.

### 파일 구조

```
.claude/planning-plugin/
├── state.md                    # 현재 활성 세션 정보
└── sessions/
    ├── 20240115-143022/        # 세션 1 (완료)
    │   ├── results.md          # 불합격 아이디어 로그
    │   ├── passed/             # 합격 아이디어 (개별 파일)
    │   │   ├── IDEA-001-앱이름.md
    │   │   └── ...
    │   └── refined.md          # 고도화된 최종 아이디어
    └── ...
```

### 세션 동작

| 상황 | 동작 |
|------|------|
| `/idea-generate` 처음 실행 | 새 세션 폴더 생성 |
| 루프 진행 중 | 같은 세션 폴더에서 계속 |
| 루프 완료 후 다시 실행 | 새 세션 폴더 생성 |
| `/cancel-idea-loop` | 현재 세션 종료, 결과는 보존 |
