# 2026.01.26 기획 문서 체계 - 상위/상세 기획서 및 Planning Plugin 가이드

## 1. 기획 문서 체계

### 1.1 상위 기획서

기획의 가치를 전달하는 문서입니다. 왜 이 서비스를 만들어야 하는지, 어떤 문제를 해결하는지를 설명합니다.

- 개요: 문제 정의 + 솔루션
- 구현 개요: 서비스 구성, 주요 기능, 기능 흐름
- 타겟 유저 및 페르소나
- 핵심 지표 (KPI/Goals)
- 레퍼런스 및 벤치마킹
- (+) 기술 검토

### 1.2 상세 기획서

아이디어 구현에 초점이 맞춰진 문서입니다. UX 관점에서 최대한 빠르게 아이디어를 구체화하고 검증하는 것이 목표입니다.

- 유저플로우
- 화면별 정의: 화면 구성, 모달, 문구, 디자인 효과
- (+) 기술 검토

### 1.3 상세 정책서

아이디어 디테일에 초점이 맞춰진 문서입니다. 품질 관점에서 구현된 아이디어에 부족한 부분들을 채우는 것이 목표입니다.

- 서비스 정책
- 화면별 정책: 제한 사항, 예외 케이스, 컴포넌트 생명주기

### 1.4 기술 검토

기술 검토는 별도로 진행하며, 다음 사항을 포함합니다:

- T/F (기술 가능 여부): 안되면 대안 제시
- 참조 URL 함께 제시
- 회사 사정을 이해할 수 있는 맥락 포함

## 2. 명령어 체계 및 구현 현황

각 기획 문서 유형에 대응하는 4개의 명령어로 구성됩니다.

### 명령어 목록

| 명령어 | 용도 | 상태 |
|--------|------|------|
| `/high-level-plan` | 상위 기획서 생성 | ✅ 구현 완료 |
| `/detailed-plan` | 상세 기획서 생성 | 🔜 예정 |
| `/policy-plan` | 상세 정책서 생성 | 🔜 예정 |
| `/tech-review` | 기술 검토 생성 | 🔜 예정 |

### /high-level-plan 상세 (구현 완료)

상위 기획서 생성을 자동화하는 명령어입니다. 문답형으로 구현 개요를 수집하고, HyperBrowser를 활용하여 나머지 섹션을 자동으로 생성합니다.

**핵심 기능:**
- 문답형 구현 개요 수집: 서비스 구성, 주요 기능, 기능 흐름을 질문으로 수집
- HyperBrowser 자동 조사: 타겟 유저, KPI, 레퍼런스를 웹 검색으로 자동 생성
- 출처 URL 자동 첨부: 자동 생성된 내용에 출처 표기
- 검토 및 수정: 초안 작성 후 사용자 피드백 반영

## 3. 사용 방법

### 설치

```bash
# 플러그인 클론
git clone <repository-url> planning-plugin

# Claude Code에 플러그인 등록
claude plugins add /path/to/planning-plugin

# 설치 확인
/plugins
```

### 상위 기획서 생성 (현재 사용 가능)

```bash
# 기본 사용
/high-level-plan [프로젝트명 또는 아이디어]

# 예시
/high-level-plan 반려동물 헬스케어 앱
/high-level-plan AI 기반 영어 학습 서비스
```

### 향후 추가 예정 명령어

```bash
# 상세 기획서 생성 (예정)
/detailed-plan [프로젝트명]

# 상세 정책서 생성 (예정)
/policy-plan [프로젝트명]

# 기술 검토 생성 (예정)
/tech-review [프로젝트명]
```
